# GitHub 个人主页仓库协作规则

## 任务开始前

任何任务执行前，先完整阅读：

1. `/Users/jin/my-profile/README.md`。
2. 本仓库 `README.md`。
3. 本文件或同目录的 `CLAUDE.md`。

阅读后检查 Git 状态，再开始修改。

## 公开边界

- 本仓库是公开的 GitHub Profile README 仓库，仓库名需要保持为 `goodjin`。
- 只放公开简介、技术方向、精选项目、公开链接、公开联系方式和展示素材。
- 不得写入手机号、住址、证件、薪资、求职记录、岗位专用简历、登录信息、Cookie、Token、密钥或未公开经历。
- 从 `website/`、`linkedin/` 或 `boss/` 提取内容时，只使用已经确认可以公开的事实，并按 GitHub 读者需要重新组织。

## 修改与验证

- 保持 README 简洁，优先展示当前技术方向和可核实项目。
- 新增项目链接、指标或身份描述前核对来源和可访问性。
- 图片放在 `assets/`；不提交 `.DS_Store` 和临时文件。
- 提交前检查 README 渲染、链接和完整 diff，确保没有敏感信息。
- 完成修改后在本仓库单独提交并推送。

## 规则同步

`AGENTS.md` 与 `CLAUDE.md` 内容应保持一致；修改其中一份时同步修改另一份，并在完成前比较内容。
